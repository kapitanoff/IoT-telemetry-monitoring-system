# Chicken Monitor

Chicken Monitor -- экспериментальный стенд для приема и визуализации телеметрии IoT-датчиков через MQTT с контролем качества доставки сообщений. Датчики или программный эмулятор публикуют температуру и напряжение питания в MQTT-брокер, backend принимает сообщения с заданным уровнем MQTT QoS, сохраняет телеметрию и рассчитывает задержку, p95 latency и потери сообщений температурного потока по `seq_id`.

Интерфейс в этом проекте нужен не как отдельное frontend-приложение, а как прикладной слой наблюдения за телеметрией и состоянием канала связи: на одной панели показываются показатели птиц, актуальность датчиков и QoS-метрики передачи.


## Что проверяет система

- Принимает MQTT-сообщения от беспроводных датчиков или программного эмулятора
- Поддерживает локальный стенд с Mosquitto и режим подключения к внешнему MQTT-брокеру
- Позволяет задавать уровень MQTT QoS для проверки влияния качества доставки на задержку и потери
- Рассчитывает QoS-метрики по температурному потоку: среднюю задержку, p95 latency, packet loss и число принятых сообщений
- Использует `sent_ts` и `seq_id` в payload MQTT-сообщений для оценки задержки и пропусков последовательности
- Показывает температуру каждой курицы на панели с цветовой индикацией:
  - **Зеленый** -- температура в норме (40-42 C)
  - **Желтый** -- повышенная температура, стоит обратить внимание (42-43 C)
  - **Красный** -- опасно, курица может быть больна (ниже 40 или выше 43 C)
  - **Серый** -- статус не определен: для датчика еще нет актуального температурного значения
- Строит графики температуры за любой период -- от 1 часа до 1 года
- Показывает заряд батареи каждого датчика
- Позволяет группировать кур по загонам
- Данные обновляются автоматически каждые 3 секунды

## Режимы запуска

Репозиторий поддерживает два варианта работы:

- **Локальный экспериментальный стенд** -- используется для демонстрации и экспериментов на одном ПК. Docker Compose поднимает `postgres`, `backend` и локальный `mosquitto`, а профиль `emulator` добавляет программный эмулятор датчиков.
- **Подключение к внешнему MQTT-брокеру** -- используется, когда брокер и реальные датчики уже существуют отдельно, а проекту нужно только принимать данные и показывать их в интерфейсе.

Файл `project/docker-compose.yml` содержит базовые сервисы `postgres` и `backend`, а `project/docker-compose.override.yml` автоматически добавляет локальный `mosquitto` и профиль `emulator`. Из-за этого локальный стенд работает даже если основной compose-файл выглядит неполным.

## Что нужно для работы

- Компьютер с **Windows 10/11**, **Linux** или **Mac**, который будет работать как сервер
- Для **локального экспериментального стенда** достаточно установленного Docker Desktop
- Для **варианта с внешним брокером** нужен Docker Desktop, MQTT-брокер (например Mosquitto, EMQX, HiveMQ) и подключенные к нему датчики

## Установка

### Шаг 1. Установите Docker

Скачайте Docker Desktop и установите как обычную программу:

- **Windows 10/11:** https://docs.docker.com/desktop/setup/install/windows-install/
- **Mac:** https://docs.docker.com/desktop/setup/install/mac-install/
- **Linux:** https://docs.docker.com/engine/install/

После установки **перезагрузите компьютер**. Убедитесь, что Docker работает -- его иконка (кит) должна быть в трее (область рядом с часами).

### Шаг 2. Скачайте проект

Если на компьютере установлен Git:

```
git clone <адрес-репозитория>
cd CHICKEN-MONITORS
```

Или скачайте ZIP-архив с проектом, распакуйте в любую папку.

### Шаг 3. Выберите режим запуска

Откройте **PowerShell** (найдите через поиск в меню Пуск). Нужно перейти в папку `project` внутри проекта.

**Как узнать путь к папке:**

1. Откройте в Проводнике папку `CHICKEN-MONITORS`, затем зайдите в `project`
2. Нажмите на адресную строку сверху (где написан путь) -- он выделится
3. Скопируйте его (Ctrl+C)

В PowerShell введите `cd `, вставьте скопированный путь (правой кнопкой мыши) и нажмите Enter. Например:

```powershell
cd C:\Users\Иван\Desktop\CHICKEN-MONITORS\project
```

#### Вариант A. Локальный экспериментальный стенд

Этот режим использовался для экспериментов и демонстрации на одном ПК. В нем MQTT-брокер `mosquitto` запускается локально в Docker, а при необходимости можно добавить встроенный эмулятор датчиков.

Создайте файл `project/.env` со значениями для локального стенда:

```dotenv
MQTT_HOST=mosquitto
MQTT_PORT=1883
MQTT_USERNAME=chicken
MQTT_PASSWORD=chicken
MQTT_QOS=1

POSTGRES_USER=chicken
POSTGRES_PASSWORD=chicken123
POSTGRES_DB=chicken_monitor

EMU_SENSORS=10
EMU_QOS=1
EMU_INTERVAL=5
EMU_LOSS=0

TEMP_GREEN_MIN=40.0
TEMP_GREEN_MAX=42.0
TEMP_YELLOW_MAX=43.0
```

Запустите локальный стенд:

```powershell
docker compose up -d --build
```

Если нужен встроенный эмулятор датчиков, добавьте профиль `emulator`:

```powershell
docker compose --profile emulator up -d --build
```

В этом режиме `docker-compose.override.yml` подключается автоматически: он добавляет локальный брокер `mosquitto`, а профиль `emulator` запускает контейнер с публикацией тестовых MQTT-сообщений.

#### Вариант B. Подключение к внешнему MQTT-брокеру

Если брокер уже существует отдельно, используйте setup-скрипт. Он помогает создать `.env` для подключения к внешнему MQTT-узлу и запускает систему.

Запустите скрипт:

```powershell
.\setup.ps1
```

Скрипт задаст несколько вопросов:

1. **Адрес MQTT-брокера** -- укажите сетевой адрес, доступный из контейнера `backend`. Если брокер на другом сервере, это может быть IP-адрес вроде `192.168.1.50`. Если брокер работает на том же компьютере вне Docker, используйте адрес хоста, доступный из контейнера, например `host.docker.internal` на Windows и Mac
2. **Порт брокера** -- обычно `1883`, просто нажмите Enter
3. **Логин и пароль MQTT** -- те же, что используются для подключения датчиков к брокеру
4. **Пароль базы данных** -- нажмите Enter, скрипт сгенерирует автоматически
5. **Пороги температуры** -- нажмите Enter для значений по умолчанию (40-42-43 C)

После ответов скрипт спросит, запустить ли систему сейчас -- нажмите Enter (да).

Первый запуск займет **2-5 минут** -- система скачивает необходимые компоненты. Все последующие запуски -- несколько секунд.

**Linux / Mac:** вместо `.\setup.ps1` используйте `./setup.sh`

### Шаг 4. Откройте в браузере

Перейдите по адресу:

```
http://localhost:8000
```

Если запущен локальный стенд с профилем `emulator`, карточки появятся автоматически. Если используется внешний брокер, карточки появятся после того, как реальные датчики начнут публиковать данные в MQTT-топики.

## Как пользоваться

### Главный экран

На главном экране отображаются карточки кур. Каждая карточка показывает:

- Номер курицы (совпадает с номером датчика)
- Текущую температуру
- Значок батареи датчика
- Цветовой статус (зеленый / желтый / красный / серый)

Карточки отсортированы по срочности: сначала красные (требуют внимания), потом желтые, затем серые со статусом неопределенности, затем зеленые.

В верхней части экрана показано общее количество кур по статусам. Нажмите на число -- отфильтруются только курицы с этим статусом.

### Графики температуры

Нажмите на карточку курицы -- откроется окно с графиком температуры. Сверху можно выбрать период: 1 час, 6 часов, 24 часа, 7 дней, 30 дней, 6 месяцев, 1 год.

Под графиком отображаются минимальная, максимальная и средняя температура за период.

Стрелками влево/вправо можно переключаться между курицами, не закрывая окно.

### Загоны (группировка)

Чтобы не искать нужную курицу среди сотен карточек, их можно разделить по загонам:

1. Нажмите **Загоны** в шапке -- откроется окно управления
2. Введите название загона (например "Корпус 1") и нажмите **Создать**
3. Нажмите **По загонам** -- карточки разделятся по группам

**Назначить курицу в загон:**

- Нажмите на карточку курицы, выберите загон из списка внизу

**Назначить сразу много кур:**

1. Нажмите **Выбрать** в шапке
2. Отметьте нужных кур (или нажмите галочку у заголовка загона -- выберутся все в нем)
3. Выберите загон из списка внизу экрана
4. Нажмите **Назначить**

### Настройка порогов температуры

Нажмите значок шестеренки в шапке. Здесь задается, какая температура считается нормальной, а какая опасной:

- **Норма от** -- ниже этого значения температура опасно низкая (по умолчанию 40 C)
- **Норма до** -- выше этого значения появляется предупреждение (по умолчанию 42 C)
- **Предупреждение до** -- выше этого значения температура опасно высокая (по умолчанию 43 C)

Нажмите **Сохранить** -- карточки сразу перекрасятся по новым порогам.

## Настройки (.env)

Все настройки находятся в файле `project/.env`.

Файл `.env.example` и setup-скрипты ориентированы на вариант подключения к внешнему брокеру. Для локального экспериментального стенда значения отличаются прежде всего параметрами `MQTT_HOST=mosquitto`, `MQTT_USERNAME=chicken` и `MQTT_PASSWORD=chicken`.

### Подключение к брокеру

| Настройка | Что значит | Пример |
|-----------|-----------|--------|
| `MQTT_HOST` | Адрес MQTT-брокера | `mosquitto` для локального стенда, `192.168.1.50` для внешнего брокера |
| `MQTT_PORT` | Порт брокера | `1883` |
| `MQTT_USERNAME` | Логин для подключения к брокеру | `chicken` |
| `MQTT_PASSWORD` | Пароль для подключения к брокеру | задается при установке |
| `MQTT_QOS` | Надежность доставки: 0 (быстро), 1 (надежно), 2 (гарантированно) | `1` |

### База данных

| Настройка | Что значит | По умолчанию |
|-----------|-----------|--------------|
| `POSTGRES_PASSWORD` | Пароль базы данных (только внутренний) | задается при установке |
| `POSTGRES_USER` | Имя пользователя БД | `chicken` |
| `POSTGRES_DB` | Название базы данных | `chicken_monitor` |

### Пороги температуры

| Настройка | Что значит | По умолчанию |
|-----------|-----------|--------------|
| `TEMP_GREEN_MIN` | Нижняя граница нормальной температуры | `40.0` |
| `TEMP_GREEN_MAX` | Верхняя граница нормальной температуры | `42.0` |
| `TEMP_YELLOW_MAX` | Граница опасной температуры | `43.0` |

После изменения `.env` перезапустите систему:

```
docker compose down
docker compose up -d
```

## Подключение датчиков

Датчики должны быть подключены к тому же MQTT-брокеру, адрес которого указан в `.env`.

Формат сообщений, которые датчики отправляют брокеру:

**Топик температуры:**

```
ThermoChicken/<номер_датчика>/Temperature
```

**Топик напряжения батареи:**

```
ThermoChicken/<номер_датчика>/voltage
```

**Основной формат данных** -- JSON. Он используется эмулятором и позволяет серверу считать задержку доставки и потери сообщений:

```json
{
  "value": 41.2,
  "sent_ts": 1711464000.123,
  "seq_id": 42
}
```

Поле `value` содержит измеренное значение, `sent_ts` -- метку времени отправки, а `seq_id` -- порядковый номер сообщения.

Для обратной совместимости backend принимает и упрощенный legacy-формат с одним числом:

```text
41.2
```

## Установка на Windows 7

Docker Desktop не работает на Windows 7, поэтому система запускается через виртуальную машину VirtualBox.

1. Установите [VirtualBox 6.1](https://www.virtualbox.org/wiki/Download_Old_Builds_6_1)
2. Установите [Git 2.46.2](https://github.com/git-for-windows/git/releases/download/v2.46.2.windows.1/Git-2.46.2-64-bit.exe) (нужен для служебных программ)
3. Создайте файл настроек:
   ```powershell
   cd project
   .\setup-win7.ps1
   ```
4. Запустите виртуальную машину:
   ```powershell
   cd vm
   .\deploy-vbox.ps1
   ```
5. Откройте в браузере: **http://localhost:8080**

## Установка через Hyper-V (Windows 10/11 Pro)

Если на компьютере нельзя установить Docker Desktop, можно запустить систему в виртуальной машине Hyper-V.

1. Включите Hyper-V. Откройте PowerShell **от имени администратора** и выполните:
   ```powershell
   Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V-All
   ```
   Компьютер перезагрузится.

2. Установите QEMU (нужен для создания образа диска):
   ```powershell
   winget install SoftwareFreedomConservancy.QEMU
   ```

3. Создайте файл настроек:
   ```powershell
   cd project
   .\setup.ps1
   ```

4. Запустите виртуальную машину:
   ```powershell
   cd vm
   .\deploy.ps1
   ```

5. Откройте в браузере: **http://localhost:8000**

Для обновления системы используйте:
```powershell
cd vm
.\update.ps1
```

## Решение проблем

### Страница не открывается (localhost:8000)

1. Проверьте, что Docker Desktop запущен (иконка кита в трее)
2. Откройте терминал, перейдите в папку `project` и выполните:
   ```
   docker compose ps
   ```
   В локальном стенде обычно должны быть видны `postgres`, `backend`, `mosquitto` и, если запускался профиль, `emulator`. Их статус должен быть `running` или `healthy`
3. Если контейнер `backend` упал -- посмотрите ошибку:
   ```
   docker compose logs backend
   ```

### Карточки не появляются

- Для локального стенда проверьте, что в `.env` указан `MQTT_HOST=mosquitto`
- Для внешнего брокера проверьте, что в `.env` указан сетевой адрес, доступный из контейнера `backend`, а не адрес `localhost` внутри самого контейнера
- Проверьте, что датчики включены и подключены к тому же брокеру
- Проверьте, что логин и пароль в `.env` совпадают с учетными данными брокера
- Посмотрите логи на ошибки подключения:
  ```
  docker compose logs backend
  ```
  Если видите `MQTT connection failed` или `MQTT reconnect failed` -- проблема с адресом или паролем брокера

### Данные приходят с большой задержкой

- Проверьте Wi-Fi сигнал в месте установки датчиков
- Попробуйте уменьшить количество датчиков на один роутер
- В `.env` попробуйте `MQTT_QOS=0` (быстрее, но менее надежно)

### Как обновить систему

```
cd project
docker compose down
git pull
docker compose up -d --build
```
